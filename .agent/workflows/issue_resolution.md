---
description: Standard procedure for resolving GitHub issues
---
1. Post the To Do and its working hour estimation to the GitHub issue.
2. Find/Create a git branch aligned with this issue (e.g., `issue-101-fix-description`).
3. Update and fix the issue in the codebase.
4. Post a report of the work done to the GitHub issue.
5. Commit changes and create a pull request to the `release` branch.
